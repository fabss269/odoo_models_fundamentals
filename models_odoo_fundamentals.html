Odoo ORM — Referencia de tipos de datos

Cómo se definen campos, relaciones y opciones en un modelo de Odoo.

Tipos de datos básicos (fields)
Char Texto corto
Cadena de texto. Equivale a VARCHAR en SQL.
name = fields.Char(
    string='Nombre', required=True
)
Text Texto largo
Texto sin límite. Equivale a TEXT en SQL.
descripcion = fields.Text(
    string='Descripción'
)
Integer Número entero
Número entero sin decimales.
cantidad = fields.Integer(
    string='Cantidad', default=0
)
Float Número decimal
Decimal. Con digits controlas la precisión.
precio = fields.Float(
    string='Precio', digits=(10, 2)
)
Boolean Verdadero / Falso
Checkbox en la vista. Default es False.
activo = fields.Boolean(
    string='Activo', default=True
)
Selection Lista de opciones
Dropdown con valores fijos.
estado = fields.Selection([
    ('borrador', 'Borrador'),
    ('aprobado', 'Aprobado'),
], default='borrador')
Date Fecha
Solo fecha, sin hora. Usa fields.Date.today como default.
fecha = fields.Date(
    string='Fecha',
    default=fields.Date.today
)
Datetime Fecha + Hora
Fecha y hora completa. Se almacena en UTC.
created_at = fields.Datetime(
    default=fields.Datetime.now
)
Binary Archivo / Imagen
Datos binarios. Útil para adjuntos o imágenes.
imagen = fields.Binary(
    string='Imagen'
)
imagen_name = fields.Char()
Html Texto enriquecido
Texto con editor WYSIWYG (negritas, listas, etc.).
notas = fields.Html(
    string='Notas'
)
Campos relacionales
Many2one
Muchos registros apuntan a uno solo. El más común. Es la FK real en la tabla.
partner_id = fields.Many2one('res.partner', string='Contacto')
One2many
Uno tiene muchos. Inverso del Many2one. No crea columna en BD, es virtual.
formularios = fields.One2many('modulo.formulario', 'partner_id')
Many2many
Muchos a muchos. Crea tabla intermedia automáticamente.
etiquetas = fields.Many2many('res.partner.category')
Opciones comunes en cualquier field
string=
Etiqueta visible en la UI
required=True
Campo obligatorio (NOT NULL)
default=
Valor por defecto al crear
readonly=True
Solo lectura en la vista
help=
Tooltip informativo
index=True
Crea índice en BD (mejora búsquedas)
copy=False
No se copia al duplicar registro
tracking=True
Registra cambios en el chatter
compute=
Calculado con función Python
Campos que Odoo agrega automáticamente (no los declares)
id
Primary key autoincremental
create_date
Fecha de creación del registro
write_date
Fecha de última modificación
create_uid
Usuario que creó el registro
write_uid
Usuario que modificó por último
Estructura mínima de un modelo
from odoo import models, fields

class MiFormulario(models.Model):
    _name        = 'mi_modulo.formulario'  # nombre de la tabla en BD
    _description = 'Formulario'

    # Odoo agrega id, create_date, write_date, create_uid, write_uid automáticamente
    name       = fields.Char(string='Título', required=True)
    partner_id = fields.Many2one(
        'res.partner',
        string='Contacto',
        required=True,
    )
Tu tabla formulario (punto de partida)
Un contacto (res.partner) → muchos formularios — relación Many2one
from odoo import models, fields

class Formulario(models.Model):

    _description = 'Formulario'

    name       = fields.Char(string='Nombre', required=True)
    partner_id = fields.Many2one(

Convención de nombres
# _name usa puntos → Odoo los convierte a guiones bajos en PostgreSQL
'mi_modulo.formulario'  →  tabla mi_modulo_formulario  en la BD
'estate.property'       →  tabla estate_property        en la BD

# Convención de campos relacionales: modelo + _id
partner_id   # → Many2one a res.partner
user_id      # → Many2one a res.users
company_id   # → Many2one a res.company
